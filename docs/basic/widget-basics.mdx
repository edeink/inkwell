Inkwell 的 Widget 是构建界面的基础单元，采用 Flutter 风格的声明式布局与约束模型。你通过组合不同类型的 Widget 构建出层次清晰的布局树。

## Widget 有哪些作用
- 布局：负责元素的排列、对齐与空间分配（如行/列、居中、层叠）。
- 容器：提供尺寸限制、内外边距与背景（如通用容器、占位盒）。
- 媒体与文本：展示文本/图片等内容。
- 定位与层叠：在同一层内控制子元素的绝对位置与覆盖关系。

## 分类与对应组件

Inkwell 提供了丰富的内置组件，涵盖布局、绘制、交互等多个方面。详细 API 请参考 [API 文档](../api)。

### 1. 基础布局
- **弹性布局**: [`Row`](../api/widgets/Row) (水平)、[`Column`](../api/widgets/Column) (垂直)、[`Expanded`](../api/widgets/Expanded) (填充剩余空间)
- **层叠布局**: [`Stack`](../api/widgets/Stack)、[`Positioned`](../api/widgets/Positioned)
- **对齐与定位**: [`Center`](../api/widgets/Center)、`Align` (暂未文档化)
- **流式布局**: [`Wrap`](../api/widgets/Wrap)

### 2. 容器与绘制
- **通用容器**: [`Container`](../api/widgets/Container) (集成了尺寸、边距、背景等)
- **空间控制**: [`SizedBox`](../api/widgets/SizedBox) (固定尺寸)、[`Padding`](../api/widgets/Padding) (内边距)
- **内容展示**: [`Text`](../api/widgets/Text)、[`Image`](../api/widgets/Image)

### 3. 滚动与视口
- **滚动视图**: [`ScrollView`](../api/widgets/ScrollView)
- **视口控制**: [`Viewport`](../api/widgets/Viewport)

## 核心布局模型（简要）
- **Flex（弹性布局）**: `Row/Column` 按主轴/交叉轴对齐；子元素可通过 `Expanded` (`flex`) 参与剩余空间分配。
- Alignment（对齐）：`Center/Stack` 控制子元素在父容器中的对齐位置。
- Constraints（约束）：`Container/SizedBox` 提供固定或紧约束；`Padding` 通过内边距缩减可用空间。

## children 支持类型标识
- 不支持嵌套：如 `Text`、`Image`。
- 支持单 children：如 `Center`、`Padding`、`SizedBox`、`Container`、`Positioned`。
- 支持多 children：如 `Row`、`Column`、`Stack`。

## 常见布局模式（选择合适的组件）
- 水平排列并控制间距/对齐：使用 `Row`。
- 垂直排列并支持子元素按权重分配高度：使用 `Column` 与子元素的 `flex`。
- 同时水平/垂直居中：使用 `Center`。
- 层叠与绝对定位：使用 `Stack`，必要时配合 `Positioned`。
- 设定固定尺寸或制造空白：使用 `SizedBox`。
- 设置背景色、边框与内外边距：使用 `Container` 或 `Padding`。
