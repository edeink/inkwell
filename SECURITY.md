# 安全策略

## 支持范围

本仓库目前处于快速迭代阶段，我们会优先修复以下类型的问题：

- 远程代码执行、任意文件读写、权限绕过等高危漏洞
- 明显可被利用的数据泄露（例如敏感信息意外暴露到日志/构建产物）
- 供应链风险（依赖投毒、恶意脚本等）

## 漏洞报告方式

请不要在公开 Issue 中直接披露安全漏洞细节。

- 优先方式：使用 GitHub 的 Security Advisories 进行私密报告（仓库页面 → Security → Report a vulnerability）
- 若仓库未开启该入口：请新建 Issue，但仅描述影响范围与复现线索，避免提交可直接利用的 PoC、密钥或生产环境信息

## 我们会做什么

- 收到报告后尽快确认并评估影响范围
- 在修复完成后发布公告与更新说明（必要时附 CVE/致谢）
